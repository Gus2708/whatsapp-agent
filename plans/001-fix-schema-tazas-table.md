## Why this matters

`supabase_schema.sql` is the file the README tells a new operator to run in the
Supabase SQL editor. But it creates the exchange-rate table as `tasas` and ships
a migration block that **renames `tazas` → `tasas`**. The live production
database has only `tazas` (verified: 2 rows, no `tasas` table), and **every
piece of bot code queries `tazas`** (`/rest/v1/tazas?nombre=eq.actual`). So
running this schema against production would rename the live table out from
under the bot, and every bolívar (Bs) price would silently fall back to "tasa
BCV no disponible". This is a loaded gun in the repo. The fix aligns the schema
file with the production reality (`tazas` is canonical — confirmed by the git
history commit `34c93e6 … revert tabla tazas`).

## Current state

- `supabase_schema.sql` — the DDL a new operator runs. Two problem spots:

  Section 3 (lines 22–32) creates the table under the **wrong** name `tasas`:
  ```sql
  -- 3. Tabla de tasas de cambio dinámicas (Bolívares o pesos locales por Dólar)
  --    NOTA: tabla renombrada de "tazas" a "tasas" (corrección ortográfica)
  CREATE TABLE IF NOT EXISTS public.tasas (
      id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      nombre text,
      bcv_usd numeric,
      bcv_eur numeric,
      binance_p2p numeric,
      tasa_promedio numeric,
      created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
  );
  ```

  Section 11 (lines 178–189) renames the live table, which **breaks the bot**:
  ```sql
  -- 11. Migración: renombrar tabla tazas → tasas (ejecutar solo si existe la tabla vieja)
  -- NOTA: ejecutar este bloque SOLO si se está migrando desde una instalación anterior.
  -- En instalaciones nuevas, la tabla ya se crea como "tasas" (paso 3 arriba).
  DO $$
  BEGIN
    IF EXISTS (SELECT 1 FROM information_schema.tables WHERE table_schema = 'public' AND table_name = 'tazas')
       AND NOT EXISTS (SELECT 1 FROM information_schema.tables WHERE table_schema = 'public' AND table_name = 'tasas')
    THEN
      ALTER TABLE public.tazas RENAME TO tasas;
      RAISE NOTICE 'Tabla "tazas" renombrada a "tasas" exitosamente.';
    END IF;
  END $$;
  ```

- Proof the bot depends on `tazas` (do **not** modify these — they are correct):
  - `scratch_live/live_buscar.js:106` — `axios.get(SB+'/rest/v1/tazas?nombre=eq.actual&select=bcv_usd' ...)`
  - `scratch_live/live_presupuesto.js:99` — same `tazas` query.
  - `n8n_workflow.json` `obtener_tasa_bcv` node — `/rest/v1/tazas?nombre=eq.actual`.

- Production truth (already verified against the live DB; you do not need DB
  access to do this plan): the table is `public.tazas` with 2 rows; there is no
  `public.tasas`.

In `supabase_schema.sql` Section 3, change `CREATE TABLE IF NOT EXISTS public.tasas (`
